01 · Typeface Selection

Why Inter?

After evaluating several modern, neutral typefaces against the keywords modern and simple, Inter emerged as the clear choice for Drift.

Screen Optimised

Inter was designed specifically for display on digital screens at small sizes. Its tall x-height, wide apertures, and careful letter spacing make it exceptionally legible at the 10–24 px range used throughout the app.

Modern & Neutral

Geometric construction gives Inter a clean, contemporary feel without leaning into the quirks of humanist faces. It pairs naturally with Drift's minimal, dark UI and does not compete with content.

Weight Range

With 9 weights (100–900), Inter supports Drift's full hierarchy — from the tiny badge counts on workspace chips to the bold settings header — using a single coherent face.

Open Source & Reliable

Released under the SIL Open Font Licence. Bundled via @expo-google-fonts/inter and loaded at app start, so it works offline and requires no external network calls at runtime.

07 · Do & Don't

Rules for consistent type

✓ Do

Always use the named Inter variant (Inter_700Bold) as the fontFamily. This guarantees the correct weight on both iOS and Android without relying on the platform's font synthesis.

✗ Don't

Don't combine a named Inter variant with a fontWeight override. Font weight synthesis by the OS can produce blurry or unexpected results when the explicit file name already encodes the weight.

✓ Do

Respect the established size ramp. Use typography.fontSize.* tokens imported from the theme to keep sizes consistent and easy to change globally.

✗ Don't

Don't introduce arbitrary font sizes (e.g., 9 dp, 17 dp, 19 dp) outside the defined scale. New sizes should be proposed as additions to the typography.fontSize token set.

✓ Do

Use tight letter-spacing (−0.3) for large headings (22 dp +) to give them presence, and wide letter-spacing (0.8) for uppercase micro-labels to improve legibility.

✗ Don't

Don't use Inter in italic for UI strings — italics are not loaded in the app bundle and would fall back to system synthesis. Reserve italics for inline web content in the web view only.
